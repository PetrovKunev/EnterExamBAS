Тема 7: Езици за програмиране — классификация, парадигми

Обзорна кутия

Свързани теми:

  • Тема 3 (Алгоритми): Езиците са средство за изпълнение на алгоритми. Различните парадигми пряко влияят на начина, по който описваме алгоритмични решения.

Раздел 1: Класификация на езиците

1.1 По ниво на абстракция (Generations)

Езиците могат да се класифицират според нивото на абстракция, което осигуряват спрямо хардуера:

1GL (Machine Code): Екзекутабилни инструкции за процесора. Директна манипулация на битове и регистри. Пример: 10110000 01100001
2GL (Assembly Language): Символни кодове за машинни инструкции. Директна работа със регистри и памет. Примери: x86 Assembly, ARM Assembly. Изисква асемблер за превод.
3GL (High-Level Languages): Абстракция от хардуерната архитектура. Примери: C, Java, Python, C++. Изискват компилатор или интерпретатор.
4GL (Very High-Level Languages): Декларативни езици, фокусирани на "ЧЕ" трябва да се направи, не "КАК". Примери: SQL (заявки към БД), MATLAB (научни вычисления), Visual Basic (RAD среди).
5GL (AI-oriented Languages): Езици за логическо програмиране и автоматично решение на проблеми. Примери: Prolog, Lisp, системи за експертни познания.

1.3 По типова система (Type System)

Типовата система определя как языкът управља и проверява типовете на данните:

Статична типова система: Типовете се проверяват при компилация. C, Java, Haskell, Rust.

Динамична типова система: Типовете се проверяват при изпълнение. Python, JavaScript, Ruby, Lua.

Силна типова система: Языкът не позволява неявни преобразувания между несъвместими типове. Java, Python, Haskell.

Слаба типова система: Позволява автоматични или явни преобразувания. C, JavaScript, PHP.

Type Inference: Компилаторът автоматично дедуцира типовете. ML, Haskell, Rust.

Номинална типова еквивалентност: Два типа са еквивалентни, ако имат същото име. Java, C++.

Структурна типова еквивалентност: Два типа са еквивалентни, ако имат същата структура. Go, TypeScript (частично).

Параметричен полиморфизъм (Generics): Позволява писане на генерични кодове. Java (List<T>), C++ (templates), Rust.

1.4 По парадигма (Overview)

Парадигма е filosofски подход към програмирането. Главните парадигми са:

  • Императивна: Описва последователност на команди (как). Процедурна: C, Pascal. Обектно-ориентирана: Java, C++, Python.
  • Декларативна: Описва желаното състояние (какво). Функционална: Haskell, Lisp. Логическа: Prolog.
  • Мулти-парадигмени: Поддържат повече от една парадигма. Python, JavaScript, Scala, Kotlin.

Раздел 2: Императивна парадигма

2.1 Процедурно програмиране

Процедурното програмиране е един от най-старите стилове. Програмата е последовност от процедури (функции), които манипулират глобално състояние.

Процедура: Именуван набор от инструкции, който може да приема параметри и да връщa резултати. Примери: функции в C, подпрограми в Fortran.

Характеристики:

  • Съсредоточение върху алгоритъма (как да се реши проблемът)
  • Глобално състояние (променливи)
  • Последователно изпълнение на инструкции
  • Примери: C, Fortran, Pascal, Basic, Go (частично)

2.2 Структурно програмиране (Structured Programming)

През 1968 г. Edsger Dijkstra издава статията "Go To Considered Harmful", която инициира революция в програмирането. Структурното програмиране ограничава контролния поток до три основни структури:

Три основни структури:
  • Последователност: Инструкциите се изпълняват една след друга.
  • Избор (If-Then-Else): Условно изпълнение на блокове код.
  • Цикъл (While/For): Повторно изпълнение на блок, докато условието е истино.

Теорема на Бьом и Якопини (1966): Всяка програма, използваща безусловен скок (goto), може да се преобразува в еквивалентна програма, която използва само три структури. Това е теоретична основа на структурното програмиране.

2.3 Работен пример: Структурно vs Неструктурно

Задача: Намерете първото четно число в списък [3, 5, 8, 1, 10]. Ако го намерите, отпечатайте го и спрете.

Неструктурно (с goto):

i := 0
loop:
  if i >= length(list) goto end
  if list[i] mod 2 == 0 goto found
  i := i + 1
  goto loop
found:
  print list[i]
end:

Структурно:

found := false
i := 0
while i < length(list) and not found do
  if list[i] mod 2 == 0 then
    print list[i]
    found := true
  end if
  i := i + 1
end while

С break (модерна структурна):

for i := 0 to length(list) - 1 do
  if list[i] mod 2 == 0 then
    print list[i]
    break
  end if
end for

Последното е най-четливо и поддържаемо.

Раздел 3: Обектно-ориентирана парадигма (ООП)

3.1 Историческо развитие

Simula (1967) - Крилан Нигард и Оле-Йохан Даль. Първи език с класове и наследяване. Разработен за симуляции.
Smalltalk (1972) - Алан Кей и екипа. "Чистия" ООП език. Всичко е обект. Първа графична среда (Dynabook).
C++ (1985) - Бьёрн Страустръп. ООП разширение на C. Множественото наследяване.
Java (1991-1995) - Джеймс Гослинг. "Write Once, Run Anywhere" (WORA). JVM-базирана ООП. Подкрепа за многонишковост.

3.2 Четирите стълба на ООП

1. Капсулация (Encapsulation): Скриване на вътрешното състояние (private атрибути) зад публичен интерфейс. Ограничава достъпа до данните.
2. Наследяване (Inheritance): Дочерния клас наследява свойства и методи от родителския клас. Позволява йерархия и преизползване на код.
3. Полиморфизъм (Polymorphism): Един интерфейс, множество реализации. Позволява един метод да се поведе различно в различни контексти.
4. Абстракция (Abstraction): Скриване на сложност и експозиция на основните черти. Абстрактни класи и интерфейси.

3.3 Работен пример: Йерархия Shape

Задача: Дефинирайте йерархия от фигури (Shape) с наследяване и полиморфизъм. Всяка фигура трябва да има метод area() и display(). 
abstract class Shape {
  protected String color;

  abstract double area();
  abstract void display();
}

class Circle extends Shape {
  private double radius;

  Circle(double radius, String color) {
    this.radius = radius;
    this.color = color;
  }

  double area() {
    return Math.PI * radius * radius;
  }

  void display() {
    System.out.println("Circle: " + color + ", area=" + area());
  }
}

class Rectangle extends Shape {
  private double width, height;

  Rectangle(double width, double height, String color) {
    this.width = width;
    this.height = height;
    this.color = color;
  }

  double area() {
    return width * height;
  }

  void display() {
    System.out.println("Rectangle: " + color + ", area=" + area());
  }
}

// Употреба:
Shape[] shapes = {
  new Circle(5, "red"),
  new Rectangle(4, 6, "blue")
};

for (Shape s : shapes) {
  s.display();  // Полиморфизъм: извиква правилния метод
}

Полиморфизмът позволява итерирането през различни типове с един и същи метод.

Раздел 4: Функционална парадигма

4.1 Ламбда смятане (Lambda Calculus)

Развито от Алончо Чьорч през 1930-те години като математическа основа на функционалното програмиране.

Ламбда израз: λx.E, където x е параметър, E е тело. Пример: λx.x+1 (функция, която добавя 1 към x).
Beta-редукция: Приложение на функция към аргумент. (λx.x+1) 5 → 6
Alpha-еквивалентност: Различни имена на параметри, същата функция. λx.x+1 ≡ λy.y+1

4.2 Основни концепции

  • Чисти функции (Pure Functions): Функциите не имат странични ефекти. Същия вход дава същия изход.
  • Неизменимост (Immutability): Данните не се modificат след създаване. Нови стойности се създават чрез трансформация.
  • Higher-order функции: Функции приемат функции като параметри или връщат функции.
  • First-class функции: Функциите могат да се присвояват, предават и съхранявват.

Основни езици:

  • Lisp (1959) - Джон Маккарти. "List Processing". Първи функционален език.
  • ML (1973) - Робин Милнър. Статична типизация с type inference.
  • Haskell (1990) - Симон Пейтън Джоунс и др. "Чист" функционален език. Ленива евалуация.
  • Erlang (1986) - Джо Армстронг. За телекомуникационни системи. Конкурентност.

4.3 Работен пример: map/filter/reduce

Задача: От списък [1,2,3,4,5], получете сумата на квадратите на четните числа.

Решение в Python:

numbers = [1, 2, 3, 4, 5]

# Функционален подход:
result = sum(
  map(lambda x: x * x,
    filter(lambda x: x % 2 == 0, numbers)
  )
)

# Алтернативно с list comprehension:
result = sum(x*x for x in numbers if x % 2 == 0)

# Резултат: 2^2 + 4^2 = 4 + 16 = 20
print(result)  # 20

Решение в Haskell:

numbers = [1, 2, 3, 4, 5]

-- Функционален подход:
result = sum . map (^ 2) . filter even $ numbers

-- Резултат: 20

Обяснение на преобразуванията:

  • filter: [1,2,3,4,5] → [2,4] (четни)
  • map квадрирам: [2,4] → [4,16]
  • sum: [4,16] → 20

Раздел 5: Логическа/Декларативна парадигма

5.1 Prolog

Prolog (Programming in Logic) е базиран на математическата логика. Програмата се състои от факти и правила. Системата решава заявки чрез дедукция.

Факт: Твърдение, което е винаги истинско. Синтаксис: predicate(arg1, arg2).
Правило: Условно твърдение. Синтаксис: head :- body.
Заявка (Query): Въпрос към системата. Синтаксис: ?- goal.

5.2 Работен пример: Семейни отношения

Задача: Дефинирайте факти и правила за семейни отношения. Отговорете на заявки.

Факти:

% Родители
parent(alice, bob).
parent(alice, charlie).
parent(bob, david).
parent(bob, emma).

% Пол
male(bob).
male(charlie).
male(david).
female(alice).
female(emma).

Правила:

% X е майка на Y, ако X е родител на Y и X е женска
mother(X, Y) :- parent(X, Y), female(X).

% X е баща на Y, ако X е родител на Y и X е мъж
father(X, Y) :- parent(X, Y), male(X).

% X е дядо на Y, ако X е родител на Z и Z е родител на Y
grandparent(X, Y) :- parent(X, Z), parent(Z, Y).

% X е сестра на Y, ако имат един и същ родител и X е женска
sister(X, Y) :- parent(Z, X), parent(Z, Y), female(X), X \= Y.

Заявки и отговори:

?- mother(alice, X).
% Отговор: X = bob; X = charlie.

?- father(bob, Y).
% Отговор: Y = david; Y = emma.

?- grandparent(alice, david).
% Отговор: yes.

?- sister(X, david).
% Отговор: X = emma.

5.3 Други логически езици

  • Datalog: Подмножество на Prolog без функции. Използва се в databaze и анализа на безопасност.
  • Constraint Logic Programming (CLP): Разширение на Prolog с ограничения. Примери: CLP(FD) за целите числа, CLP(R) за реални числа.

Раздел 7: Ключови концепции в езиците

7.1 Обхват и свързване (Scope and Binding)

Статичен (лексикален) обхват: Обхватът се определя от текста на кода (структурата на вложаност). Повечето съвременни езици използват статичен обхват.
Динамичен обхват: Обхватът се определя от време изпълнение (call stack). Рядко използван; Bash, Lisp (по избор).

Пример с статичен обхват (Python):

x = "глобален"

def outer():
    x = "външен"

    def inner():
        x = "вътрешен"
        print(x)  # Отпечатва: "вътрешен"

    inner()
    print(x)  # Отпечатва: "външен"

outer()
print(x)  # Отпечатва: "глобален"

7.2 Механизми за предаване на параметри

Механизъм Описание Примери Производителност
По стойност (by value) Копира се стойността. Промени в функцията не влияят на оригинала. C, Java (примитивни типове), Python (за числа) Над памет за больших данни
По референция (by reference) Предава се адресът. Промени влияят на оригинала. C/C++ (указатели), Java (обекти), Python (мутативни обекти) Ефикасно, но рискувано
По име (by name) Параметърът се evalва всеки път, когато се ползва. Ленива евалуация. Haskell, Scheme (thunks) Подходящо за ленива евалуация
Ленива евалуация (lazy evaluation) Аргументът се не евалува, докато не е необходим. Haskell (по подразумение) Оптимизирано за безкрайни структури

7.3 Управление на паметта

Стек (Stack): LIFO памет за локални променливи и адреси на връщане. Автоматично се освобождава при излизане от функция. Бързо, но ограничено.
Куп (Heap): Произволна памет за динамични структури данни. Ръчно управление (C) или автоматично (GC). Бавно, но неограничено.

Garbage Collection (GC): Автоматично освобождаване на неупотребявана памет.

  • Mark-and-sweep: Маркира достижими обекти, след което свързва неклассифициранных.
  • Reference counting: Брой на референциите към всеки обект. Освобождава се, когато счетач е 0.
  • Generational GC: Обекти се разделят на поколения. Млади обекти се събират по-често.

Езици и управление на паметта:

  • Ръчно: C, C++ (free/delete)
  • Автоматично с GC: Java, Python, C#, Lisp, Haskell
  • Собственост (ownership): Rust (RAII, borrow checker)

Основни тенденции

  • 1950-1960: Математически основи (Fortran, Lisp, Algol). Ориентирани към напрямо изпълнение на научни算 алгоритми.
  • 1960-1980: Структурно програмиране (C), ООП (Simula, Smalltalk). Съсредоточение върху разработката на големи системи.
  • 1990-2010: Интернет и мобилност (Java, JavaScript, Python). Динамични езици за бързо прототипиране.
  • 2010-наст.: Безопасност и конкурентност (Rust, Go, Kotlin). Полиморфизъм, асинхронност, безопасност паметта.

Флашкарти за самопроверка

Кликнете върху всяка карта, за да видите отговора. Тези карти са критични за изпита.

Въпрос 1: Определете първото поколение езици (1GL) и дайте пример.
Отговор: 1GL е машинен код (binary). Директни инструкции за процесора. Пример: 10110000 01100001. Немеса развитие, но най-бързо изпълнение. [Кликнете, за да скриете]
Въпрос 2: Какво е структурното програмиране? Кой го предложи?
Отговор: Структурното програмиране е парадигма, която използва три основни структури: последователност, избор (if-then-else) и цикъл. Това е алтернатива на goto. Edsger Dijkstra го предложи през 1968 със статия "Go To Considered Harmful". [Кликнете, за да скриете]
Въпрос 3: Перечислите четирите стълба на ООП и кратко дефинирайте всеки.
Отговор:
1. Капсулация: Скриване на вътрешното състояние (private данни).
2. Наследяване: Дочерни класи наследяват от родителски.
3. Полиморфизъм: Един интерфейс, множество реализации.
4. Абстракция: Скриване на сложност, изложение на основите. [Кликнете, за да скриете]
Въпрос 4: Ламбда смятане. Какво е β-редукция?
Отговор: Ламбда смятане (λ-calculus) е математическа основа на функционално програмиране, развита от Алончо Чьорч. β-редукция е приложението на функция към аргумент. Пример: (λx.x+1) 5 → 6. Замященват формалния параметър х с аргумент 5. [Кликнете, за да скриете]
Въпрос 5: Prolog: факти, правила, заявки. Дайте един пример за правило.
Отговор:
Факт: parent(alice, bob). (неусловно твърдение)
Правило: mother(X, Y) :- parent(X, Y), female(X). (условно)
Заявка: ?- mother(alice, X). (въпрос към системата)
Правилото казва: X е майка на Y, ако X е родител на Y И X е женска. [Кликнете, за да скриете]
Въпрос 6: Разлика между статична и динамична типова система. Примери.
Отговор:
Статична: Типове се проверяват при компилация. Примери: Java, C, Rust, Haskell. Грешките се открива рано.
Динамична: Типове се проверяват при изпълнение. Примери: Python, JavaScript, Ruby. По-гъвкава, но рискувана. [Кликнете, за да скриете]
Въпрос 7: Какво е чиста функция (pure function)? Защо е важна в функционално програмиране?
Отговор: Чистата функция няма странични ефекти и зависи само от своите параметри. Същия вход дава същия изход, всеки път. Примери: f(x) = x+1. Важна е защото улеснява reasoninging, тестване и паралелизация. Противоположност: функции със странични ефекти (пишат във файл, изменяват глобално състояние). [Кликнете, за да скриете]
Въпрос 8: JVM-базирани езици vs компилирани към машинен код. Предимства и недостатъци.
Отговор:
JVM (Java, Scala, Kotlin):
+ Преносимост (WORA)
+ Гаран за безопасност памет
- По-бавен старт (инициализация на VM)
Машинен код (C, C++, Go):
+ Директно, бързо
+ Малки файлове
- Платформозависимост
- Отговорност за памет [Кликнете, за да скриете]
Въпрос 9: Какво е собственост (ownership) в Rust? Как се различава от garbage collection?
Отговор: Собствеността е система за управление на паметта в Rust. Всяка стойност има един собственик. Когато собственикът излиза от обхват, памет се освобождава (RAII). На разлика от GC, която бяга както (stop-the-world), собствеността е детерминистична и няма гауранции. [Кликнете, за да скриете]
Въпрос 10: Първи три ООП езика, които са се явили исторически. Какво е било ново в всеки?
Отговор:
1. Simula (1967): Първи ООП език с класи, наследяване и виртуални методи.
2. Smalltalk (1972): "Чистия" ООП - всичко е обект. Първа графична среда.
3. C++ (1985): ООП разширение на C. Множественото наследяване, шаблони. [Кликнете, за да скриете]
Въпрос 11: Разлика между статичен (лексикален) и динамичен обхват. Примeр.
Отговор:
Статичен (лексикален): Обхватът се определя от текста на кода. Това, което видим е това, което имаме. Java, Python, C.
Динамичен: Обхватът зависи от call stack. Рядко; Bash, Lisp (по избор).
Статичен е по-лесен за понимане и е стандартният избор на съвременни езици. [Кликнете, за да скриете]
Въпрос 12: Какво е type inference? Кои езици го поддържат?
Отговор: Type inference е способностята на компилаторът да се дедуцирам типовете на променливите без явно деклариран. Примери: ML (от 1973), Haskell, Rust, Scala, C++ (auto ключова дума). На разлика от явно дефинирани типове (int x = 5), можете да напишете (x = 5) и компилаторът го разбира. [Кликнете, за да скриете]

Бърза справка

Основни парадигми

Императивна

Суть: Как да направим нещо.

Примери: C, Java (частично), Pascal, Go.

Особено: Команди, последовательност, променливи.

Декларативна

Суть: Какво искам.

Примери: Функционална (Haskell), логическа (Prolog), SQL.

Особено: Трансформация данни, декларация на желания.

Поколенията на езиците

1GL-2GL

Машинен код и асемблер. Директна работа с хардуера. Древни, рядко се използват.

3GL

Висок ниво абстракция. C, Java, Python. Доминиращи днес.

4GL

Декларативен. SQL, MATLAB. Специализирани задачи.

5GL

AI ориентирани. Prolog, Lisp. Логическо решение на проблеми.

Ключови типове системи

Статична + силна

Java, C#, Rust, Haskell

Най-безопасна. Грешките се открива при компилация.

Динамична + силна

Python, Ruby, Scheme

Гъвкава, но грешките се открива при изпълнение.

Статична + слаба

C, C++

Контрол, но рискувано. Неявни преобразувания.

Динамична + слаба

JavaScript, PHP

Много гъвкава, но неразбираема. "Делај всичко".

Механизми за предаване на параметри

По стойност (by value)

Копира се данните. Промени не влияят на оригинала. По-безопасно, по-скъпо.

По референция (by reference)

Предава се адресът. Промени влияят. По-ефикасно, по-рискувано.
