Раздел 1: Основни понятия

Определение и основни характеристики

Граф G = (V, E): Ordenied pair where V = {v₁, v₂, ..., vₙ} is a set of vertices (nodes) and E ⊆ V × V is a set of edges (connections between vertices).

Видове графи

  • Насочен граф (Directed graph): Всяко ребро има посока (u, v) ≠ (v, u)
  • Ненасочен граф (Undirected graph): Ребра без посока; (u, v) = (v, u)
  • Претеглен граф (Weighted): Всяко ребро има тегло w(e) ≥ 0
  • Непретеглен граф (Unweighted): Всички тегла = 1
  • Ацикличен насочен граф (DAG): Насочен граф без цикли

Степени и пътища

  • Степен (Degree): Брой ребра, инцидентни на върха v
  • In-degree (indeg): Брой входящи ребра (само в насочени графи)
  • Out-degree (outdeg): Брой изходящи ребра (само в насочени графи)
  • Път (Path): Последователност от върхове (v₁, v₂, ..., vₖ), където (vᵢ, vᵢ₊₁) ∈ E
  • Прост път: Път без повтаряне на върхове
  • Цикъл (Cycle): Път, където v₁ = vₖ
  • Свързана компонента: Максимално подмножество на върхове, между които съществува път

Представяния на графи

Две основни структури за съхранение:

1. Матрица на съседствата (Adjacency Matrix)

A[i][j] = { weight(i,j), if (i,j) ∈ E; 0 or ∞, otherwise }

Пример за 4-върхов граф:

      0   1   2   3
    0 [0   1   0   4]
    1 [1   0   2   0]
    2 [0   2   0   3]
    3 [4   0   3   0]

2. Списък на съседствата (Adjacency List)

За същия граф:

    0: [1→1, 3→4]
    1: [0→1, 2→2]
    2: [1→2, 3→3]
    3: [0→4, 2→3]

Сравнение на представянията

Операция Матрица Списък Бележка
Съхранение O(V²) O(V+E) Списък е по-ефикасен за разредени графи
Проверка (u,v) O(1) O(deg(u)) Матрица е по-бърза
Намиране съседи O(V) O(deg(v)) Списък е по-бърз
Добавяне ребро O(1) O(1) Еднакво

Специални видове графи

Пълен граф (Complete graph Kₙ)

Всеки връх е свързан с всеки други връх. |E| = n(n-1)/2 за ненасочен, n(n-1) за насочен.

Двуделен граф (Bipartite graph)

V може да се дели на две непразни множества U и V, така че всяко ребро има един край в U и един в V.

Планарен граф (Planar graph)

Граф, който може да бъде начертан на равнина без пресичане на ребра.

Ключови теореми

Лема на ръкуванието (Handshaking Lemma):
Σ deg(v) = 2|E| за ненасочени графи.
За насочени: Σ indeg(v) = Σ outdeg(v) = |E|.
Формула на Euler за планарни графи:
V - E + F = 2, където V е върхове, E е ребра, F е лица.
Практично приложение: Формулата на Euler се използва за проверка на планарност. Ако граф е планарен и свързан: E ≤ 3V - 6.

Раздел 2: Обхождане на графи

Преглед на методите

Обхождането (traversal) посещава всеки връх точно веднъж. Два основни подхода:

BFS - Търсене в ширина (Breadth-First Search)

Алгоритъм

BFS(Graph G, Vertex start): queue ← new Queue() visited ← empty set queue.enqueue(start) visited.add(start) while queue is not empty: v ← queue.dequeue() process(v) for each neighbor u of v: if u not in visited: visited.add(u) queue.enqueue(u)
Сложност: O(V + E) при представяне със списък на съседствата.
Приложение: Намиране на най-кратък път в ненагласен граф, проверка на свързаност, намиране на слабо свързани компоненти.

DFS - Търсене в дълбочина (Depth-First Search)

Алгоритъм (рекурсивен)

DFS(Graph G, Vertex v, Set visited): visited.add(v) process(v) for each neighbor u of v: if u not in visited: DFS(G, u, visited) DFS-All(Graph G): visited ← empty set for each vertex v in G: if v not in visited: DFS(G, v, visited)
Сложност: O(V + E) при представяне със списък на съседствата.
Приложения: Топологично сортиране, откритие на цикли, силно свързани компоненти, мост-ребра.

Работена примера: BFS и DFS на конкретен граф

Граф с 6 върха:

graph LR A["A"] B["B"] C["C"] D["D"] E["E"] F["F"] A -->|1| B A -->|1| C B -->|1| D C -->|1| E D -->|1| F E -->|1| F

BFS от A:

  1. Начало: visited = {A}, queue = [A]
  2. Обход A: посещаване на B, C. queue = [B, C]
  3. Обход B: посещаване на D. queue = [C, D]
  4. Обход C: посещаване на E. queue = [D, E]
  5. Обход D: посещаване на F. queue = [E, F]
  6. Обход E: F вече посещен. queue = [F]
  7. Обход F: никакви нови съседи. queue = []

Ред на посещение: A, B, C, D, E, F

DFS от A:

  1. Начало: посещаване на A
  2. От A: отиваме на B (първи съсед)
  3. От B: отиваме на D (първи неопосещен)
  4. От D: отиваме на F
  5. От F: съсед E неопосещен, отиваме на E
  6. От E: всички съседи посещени
  7. Връщане и посещаване на C (съсед на A)

Ред на посещение: A, B, D, F, E, C

Топологично сортиране

Топологично сортиране: Линейно подредени всички върхове така, че за всяко ребро (u, v) във V, u идва преди v. Възможно само в DAG.

Алгоритъм на Kahn

Topological-Sort(DAG G): indegrees ← calculate in-degrees queue ← vertices with indegree 0 result ← [] while queue is not empty: v ← queue.dequeue() result.add(v) for each neighbor u of v: indegree[u]-- if indegree[u] == 0: queue.enqueue(u) return result

Пример: DAG с 5 върха

graph LR A["A (indeg:0)"] B["B (indeg:1)"] C["C (indeg:1)"] D["D (indeg:2)"] E["E (indeg:1)"] A -->|1| B A -->|1| C B -->|1| D C -->|1| D D -->|1| E

Топологично сортиране: A, B, C, D, E (или A, C, B, D, E)

Раздел 3: Минимално покриващо дърво (MST)

Минимално покриващо дърво: Подмножество от ребра, свързващо всички върхове с минимално сумарно тегло, без цикли.

Два класически алгоритма, базирани на гординска стратегия:

Алгоритъм на Kruskal

Kruskal(Graph G with weights): edges ← sort G.E by weight (ascending) mst ← [] uf ← UnionFind(G.V) for each edge (u,v,w) in edges: if uf.find(u) != uf.find(v): mst.add((u,v,w)) uf.union(u, v) return mst
Сложност: O(E log E) за сортиране + O(E α(V)) за Union-Find = O(E log E).
Подход: Гордийски - добавяме ребра в нарастващ ред на тегло, избягвайки цикли.

Алгоритъм на Prim

Prim(Graph G, Vertex start): mst ← [] visited ← {start} pq ← priority queue with edges from start while |visited| < |V|: (u, v, w) ← pq.extractMin() if v not in visited: visited.add(v) mst.add((u,v,w)) for each neighbor x of v: if x not in visited: pq.insert((v, x, weight(v,x))) return mst
Сложност: O((V+E) log V) с двоична приоритетна опашка или O(V²) с масив.
Подход: Разширяме MST от един връх, винаги добавяйки най-леко ребро до нов връх.

Работена примера: Kruskal на претеглен граф

Граф с 6 върха и 9 ребра:

graph LR A["A"] B["B"] C["C"] D["D"] E["E"] F["F"] A -->|1| B A -->|4| C B -->|2| C B -->|5| D C -->|3| D C -->|6| E D -->|7| E D -->|8| F E -->|9| F

Стъпки на Kruskal:

  1. Сортиране: (A,B,1), (B,C,2), (C,D,3), (A,C,4), (B,D,5), (C,E,6), (D,E,7), (D,F,8), (E,F,9)
  2. Стъп 1: Добавяме (A,B,1). MST={(A,B,1)}, тегло=1
  3. Стъп 2: Добавяме (B,C,2). MST={(A,B,1), (B,C,2)}, тегло=3
  4. Стъп 3: Добавяме (C,D,3). MST={(A,B,1), (B,C,2), (C,D,3)}, тегло=6
  5. Стъп 4: Пропускаме (A,C,4) - циклус с A-B-C
  6. Стъп 5: Пропускаме (B,D,5) - циклус
  7. Стъп 6: Добавяме (C,E,6). MST={(A,B), (B,C), (C,D), (C,E)}, тегло=12
  8. Стъп 7: Добавяме (D,F,8). MST завършено (5 ребра за 6 върха)

Тегло на MST: 1+2+3+6+8 = 20

Забележка за коректност: Tanto Kruskal, както и Prim, дават оптимално решение. Доказателството използва обмен на аргумент (exchange argument).

Раздел 4: Най-кратък път

Проблем на най-краткия път

Дадени: Граф G, начален връх s, целеви връх t (или всички върхове). Намерете пътя с минимално сумарно тегло.

Алгоритъм на Dijkstra

Предпоставка: Всички тегла са неотрицателни (w(e) ≥ 0).
Dijkstra(Graph G, Vertex start): dist[start] ← 0 dist[v] ← ∞ for all other v pq ← priority queue with (0, start) while pq is not empty: (d, u) ← pq.extractMin() if d > dist[u]: continue for each neighbor v of u: if dist[u] + weight(u,v) < dist[v]: dist[v] ← dist[u] + weight(u,v) pq.insert((dist[v], v)) return dist
Сложност: O((V+E) log V) с двоична приоритетна опашка. Може да бъде O(V²) с масив.

Алгоритъм на Bellman-Ford

Преимущество: Работи с отрицателни тегла и открива отрицателни цикли!
Bellman-Ford(Graph G, Vertex start): dist[start] ← 0 dist[v] ← ∞ for all other v for i = 1 to |V|-1: for each edge (u,v,w): if dist[u] != ∞ and dist[u] + w < dist[v]: dist[v] ← dist[u] + w // Проверка за отрицателни цикли for each edge (u,v,w): if dist[u] + w < dist[v]: return "Отрицателен цикъл!" return dist
Сложност: O(V·E) - по-бавно от Dijkstra, но по-гъвкаво.

Floyd-Warshall (Всички двойки)

Проблем: Намиране на най-краткия път между ВСИЧКИ двойки върхове.
Floyd-Warshall(Matrix dist): for k = 0 to |V|-1: for i = 0 to |V|-1: for j = 0 to |V|-1: dist[i][j] ← min(dist[i][j], dist[i][k] + dist[k][j]) return dist
Сложност: O(V³) - проста, но тежка за голяма V.

Работена примера: Dijkstra

Претеглен граф с 5 върха, начало A:

graph LR A["A"] B["B"] C["C"] D["D"] E["E"] A -->|2| B A -->|4| C B -->|1| C B -->|7| D C -->|3| D C -->|5| E D -->|1| E

Таблица на обновяванията:

Стъп Текущ (d) dist[A] dist[B] dist[C] dist[D] dist[E]
0 (начало) - 0 ∞ ∞ ∞ ∞
1 A(0) 0 2 4 ∞ ∞
2 B(2) 0 2 3 9 ∞
3 C(3) 0 2 3 6 8
4 D(6) 0 2 3 6 7
5 E(7) 0 2 3 6 7

Резултат: dist[A]=0, dist[B]=2, dist[C]=3, dist[D]=6, dist[E]=7

Сравнение на алгоритмите

Алгоритъм Сложност Отрицателни тегла Всички двойки
Dijkstra O((V+E) log V) ❌ Не Един източник
Bellman-Ford O(V·E) ✓ Да Един източник
Floyd-Warshall O(V³) ✓ Да ✓ Всички двойки

Раздел 5: Дървета (като графи)

Дърво като специален граф

Дърво: Свързан ненасочен граф без цикли. Еквивалентно: G е дърво iff G е свързан и |E| = |V| - 1.

Основни свойства

  • Единствен път: Между всеки две върхове в дърво съществува точно един прост път.
  • Ребра: За n върхове, има точно n-1 ребра.
  • Листо: Връх с степен 1 (в ненасочен случай).
  • Корен: В насочено дърво - един връх без входящи ребра (родител на всички).

Вкоренено дърво (Rooted Tree)

Вкоренено дърво: Дърво с выбран корен r. Всяко ребро е насочено от родител към дете.
  • Височина на дърво: Максимално разстояние от корена до листо.
  • Дълбочина на връх: Разстояние от корена до върха.
  • Поддърво: Дърво с корен като някой връх v.

Двоично дърво (Binary Tree)

Всеки върх има най-много 2 деца (ляво и дясно). Класификация:
  • Пълно двоично дърво: Всеки връх има 0 или 2 деца.
  • Перфектно двоично дърво: Всички листа на една височина.
  • Балансирано двоично дърво: Височина = O(log n).
За детайлно на двоични дървета като структури от данни, вж. Тема 5: Структури от данни.

Преходи на дърво

От гра́фова перспектива, преходите са специфични DFS/BFS реализации:

Вътрешен преход (Inorder) за двоично дърво

Inorder(Node v): if v is not null: Inorder(v.left) process(v) Inorder(v.right)

Преход на Euler тур

Euler тур: Тур по всички ребра точно веднъж. Възможен iff граф е свързан и всички върхове имат четна степен (за Euler верига: точно 2 върхове с нечетна степен).

Пример: Дърво с преход

Двоично дърво с 7 върха:

graph TD A["A"] B["B"] C["C"] D["D"] E["E"] F["F"] G["G"] A --> B A --> C B --> D B --> E C --> F C --> G

Преходи:

  • Inorder: D, B, E, A, F, C, G
  • Preorder: A, B, D, E, C, F, G
  • Postorder: D, E, B, F, G, C, A
  • BFS (ниво): A | B, C | D, E, F, G

Раздел 6: Специални задачи и проблеми

Euler пътя и верига

Euler верига (Euler circuit): Път, който преминава всяко ребро точно веднъж и се завършва в началния връх.
Euler пътя (Euler path): Път, който преминава всяко ребро точно веднъж (разпочива в един връх, завършва в друг).
Условие на Euler:
  • Euler верига съществува iff граф е свързан и ВСЕ върхове имат четна степен.
  • Euler пътя съществува iff граф е свързан и имаме ТОЧНО 2 върхове с нечетна степен.

Hamilton пътя и верига

Hamilton верига: Път, който посещава всеки връх точно веднъж и се завършва в началния.
Hamilton пътя: Път, който посещава всеки връх точно веднъж.
NP-пълнота: Проблемът на Hamilton верига е NP-пълен. Няма известен полиномиален алгоритъм.
Связь с Тема 3 (Алгоритми): Hamilton верига е класически пример за NP-пълен проблем, който демонстрира разликата между P и NP класове.

Оцветяване на граф (Graph Coloring)

Оцветяване на върхове: Назначаване на цветове на върхове така, че никой съседни върхове да няма същия цвят.
Хроматично число χ(G): Минимален брой цветове, необходими.

Гордийски алгоритъм на оцветяване

Greedy-Coloring(Graph G): color[v] ← undefined for all v for each vertex v in order: used_colors ← set of colors of neighbors of v color[v] ← smallest color not in used_colors return color
Сложност: O(V + E) за оцветяване.
Оптималност: Не е гарантирано оптимално решение, но е O(Δ+1)-приближение, където Δ е максимална степен.

Максимален поток / Минимално сечение (Max Flow / Min Cut)

Проблем: Дан е граф с капацитети на ребра. Намерете максимален поток от източник s до сифон t.

Ford-Fulkerson алгоритъм (кратко)

Ford-Fulkerson(Graph G, Vertex s, Vertex t): flow ← 0 while exists path from s to t in residual graph: f_path ← capacity of minimum edge in path flow += f_path update residual capacities return flow
Теорема на Max-Flow Min-Cut: Максимален поток = минимално сечение.

Двуделно съвпадение

Двуделно съвпадение (Bipartite Matching): Намиране на максимален набор от несвързващи се ребра между две раздели множества.

Може да се реши през алгоритъм на максимален поток с единични капацитети.

Теорема на König: В двуделен граф, размерът на максимално съвпадение = размерът на минимално върхово покритие.

Какво е граф G = (V, E)?
Наредена двойка където V е множество върхове, E ⊆ V × V е множество ребра.
Лема на ръкуванието
Сума от степени = 2|E| в ненасочен граф. За насочен: сума in-deg = сума out-deg = |E|.
Сложност на BFS/DFS?
O(V + E) при представяне със списък на съседствата.
Кога е възможна Euler верига?
Граф свързан И всички върхове имат четна степен.
Сложност Dijkstra?
O((V+E) log V) с приоритетна опашка, работи за неотрицателни тегла.
Сложност Kruskal MST?
O(E log E) за сортиране + O(E α(V)) за Union-Find = O(E log E).
Дърво: колко ребра за n върхове?
Точно n - 1 ребра. Свързано и без цикли.
Hamilton верига е...
NP-пълен проблем. Посещава всеки връх точно веднъж.
Floyd-Warshall сложност?
O(V³). За всички двойки най-кратки пътища. Работи с отрицателни тегла.
DAG определение
Directed Acyclic Graph - насочен граф без цикли. Позволява топологично сортиране.
