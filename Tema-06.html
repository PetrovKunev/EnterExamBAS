Раздел 1: Основни понятия

Определение и основни характеристики

Граф G = (V, E): Ordenied pair where V = {v₁, v₂, ..., vₙ} is a set of vertices (nodes) and E ⊆ V × V is a set of edges (connections between vertices).

Видове графи

  • Насочен граф (Directed graph): Всяко ребро има посока (u, v) ≠ (v, u)
  • Ненасочен граф (Undirected graph): Ребра без посока; (u, v) = (v, u)
  • Претеглен граф (Weighted): Всяко ребро има тегло w(e) ≥ 0
  • Непретеглен граф (Unweighted): Всички тегла = 1
  • Ацикличен насочен граф (DAG): Насочен граф без цикли

Степени и пътища

  • Степен (Degree): Брой ребра, инцидентни на върха v
  • In-degree (indeg): Брой входящи ребра (само в насочени графи)
  • Out-degree (outdeg): Брой изходящи ребра (само в насочени графи)
  • Път (Path): Последователност от върхове (v₁, v₂, ..., vₖ), където (vᵢ, vᵢ₊₁) ∈ E
  • Прост път: Път без повтаряне на върхове
  • Цикъл (Cycle): Път, където v₁ = vₖ
  • Свързана компонента: Максимално подмножество на върхове, между които съществува път

Представяния на графи

Две основни структури за съхранение:

1. Матрица на съседствата (Adjacency Matrix)

A[i][j] = { weight(i,j), if (i,j) ∈ E; 0 or ∞, otherwise }

Пример за 4-върхов граф:

      0   1   2   3
    0 [0   1   0   4]
    1 [1   0   2   0]
    2 [0   2   0   3]
    3 [4   0   3   0]

2. Списък на съседствата (Adjacency List)

За същия граф:

    0: [1→1, 3→4]
    1: [0→1, 2→2]
    2: [1→2, 3→3]
    3: [0→4, 2→3]

Сравнение на представянията

Операция Матрица Списък Бележка
Съхранение O(V²) O(V+E) Списък е по-ефикасен за разредени графи
Проверка (u,v) O(1) O(deg(u)) Матрица е по-бърза
Намиране съседи O(V) O(deg(v)) Списък е по-бърз
Добавяне ребро O(1) O(1) Еднакво

Специални видове графи

Пълен граф (Complete graph Kₙ)

Всеки връх е свързан с всеки други връх. |E| = n(n-1)/2 за ненасочен, n(n-1) за насочен.

Двуделен граф (Bipartite graph)

V може да се дели на две непразни множества U и V, така че всяко ребро има един край в U и един в V.

Планарен граф (Planar graph)

Граф, който може да бъде начертан на равнина без пресичане на ребра.

Ключови теореми

Лема на ръкуванието (Handshaking Lemma):
Σ deg(v) = 2|E| за ненасочени графи.
За насочени: Σ indeg(v) = Σ outdeg(v) = |E|.
Формула на Euler за планарни графи:
V - E + F = 2, където V е върхове, E е ребра, F е лица.
Практично приложение: Формулата на Euler се използва за проверка на планарност. Ако граф е планарен и свързан: E ≤ 3V - 6.

Раздел 2: Обхождане на графи

Преглед на методите

Обхождането (traversal) посещава всеки връх точно веднъж. Два основни подхода:

BFS - Търсене в ширина (Breadth-First Search)

Алгоритъм

BFS(Graph G, Vertex start): queue ← new Queue() visited ← empty set queue.enqueue(start) visited.add(start) while queue is not empty: v ← queue.dequeue() process(v) for each neighbor u of v: if u not in visited: visited.add(u) queue.enqueue(u)
Сложност: O(V + E) при представяне със списък на съседствата.
Приложение: Намиране на най-кратък път в ненагласен граф, проверка на свързаност, намиране на слабо свързани компоненти.

DFS - Търсене в дълбочина (Depth-First Search)

Алгоритъм (рекурсивен)

DFS(Graph G, Vertex v, Set visited): visited.add(v) process(v) for each neighbor u of v: if u not in visited: DFS(G, u, visited) DFS-All(Graph G): visited ← empty set for each vertex v in G: if v not in visited: DFS(G, v, visited)
Сложност: O(V + E) при представяне със списък на съседствата.
Приложения: Топологично сортиране, откритие на цикли, силно свързани компоненти, мост-ребра.

Детайлна примера: BFS със състояние на опашката

Граф за анализ (6 върха):

graph LR A["A"] B["B"] C["C"] D["D"] E["E"] F["F"] A -->|1| B A -->|1| C B -->|1| D C -->|1| E D -->|1| F E -->|1| F

BFS от A - Стъпка по стъпка:

Стъп Текущ връх Посетено Опашка Съседи
0 (начало) - {A} [A] -
1 A {A, B, C} [B, C] B, C от A
2 B {A, B, C, D} [C, D] D от B (A вече посещен)
3 C {A, B, C, D, E} [D, E] E от C (A вече посещен)
4 D {A, B, C, D, E, F} [E, F] F от D (B вече посещен)
5 E {A, B, C, D, E, F} [F] F вече посещен (C също)
6 F {A, B, C, D, E, F} [] D, E вече посещени

Ред на посещение: A, B, C, D, E, F (ширина-първо)

Ключово свойство: BFS посещава върховете в нарастващ ред на разстояние от начала.

Детайлна примера: DFS със състояние на стека

DFS от A - Стъпка по стъпка (използвайки същия граф):

Стъп Текущ връх Посетено Стек Действие
0 (начало) - { } [A] Инициализираме стека с A
1 A {A} [B, C] Посещаваме A, добавяме съседи B, C
2 C {A, C} [B, E] Посещаваме C, добавяме съсед E (A посещен)
3 E {A, C, E} [B, F] Посещаваме E, добавяме съсед F (C посещен)
4 F {A, C, E, F} [B, D] Посещаваме F, добавяме съсед D (E посещен)
5 D {A, C, E, F, D} [B] Посещаваме D, всички съседи посещени (B, F)
6 B {A, C, E, F, D, B} [] Посещаваме B, стекът празен

Ред на посещение: A, C, E, F, D, B (дълбочина-първо)

Ключево свойство: DFS отива възможно най-дълбоко преди да се върне назад.

Топологично сортиране

Топологично сортиране: Линейно подредени всички върхове така, че за всяко ребро (u, v) във V, u идва преди v. Възможно само в DAG.

Алгоритъм на Kahn

Topological-Sort(DAG G): indegrees ← calculate in-degrees queue ← vertices with indegree 0 result ← [] while queue is not empty: v ← queue.dequeue() result.add(v) for each neighbor u of v: indegree[u]-- if indegree[u] == 0: queue.enqueue(u) return result

Пример: DAG с 5 върха

graph LR A["A (indeg:0)"] B["B (indeg:1)"] C["C (indeg:1)"] D["D (indeg:2)"] E["E (indeg:1)"] A -->|1| B A -->|1| C B -->|1| D C -->|1| D D -->|1| E

Топологично сортиране: A, B, C, D, E (или A, C, B, D, E)

Раздел 3: Минимално покриващо дърво (MST)

Минимално покриващо дърво: Подмножество от ребра, свързващо всички върхове с минимално сумарно тегло, без цикли.

Два класически алгоритма, базирани на гординска стратегия:

Алгоритъм на Kruskal

Kruskal(Graph G with weights): edges ← sort G.E by weight (ascending) mst ← [] uf ← UnionFind(G.V) for each edge (u,v,w) in edges: if uf.find(u) != uf.find(v): mst.add((u,v,w)) uf.union(u, v) return mst
Сложност: O(E log E) за сортиране + O(E α(V)) за Union-Find = O(E log E).
Подход: Гордийски - добавяме ребра в нарастващ ред на тегло, избягвайки цикли.

Алгоритъм на Prim

Prim(Graph G, Vertex start): mst ← [] visited ← {start} pq ← priority queue with edges from start while |visited| < |V|: (u, v, w) ← pq.extractMin() if v not in visited: visited.add(v) mst.add((u,v,w)) for each neighbor x of v: if x not in visited: pq.insert((v, x, weight(v,x))) return mst
Сложност: O((V+E) log V) с двоична приоритетна опашка или O(V²) с масив.
Подход: Разширяме MST от един връх, винаги добавяйки най-леко ребро до нов връх.

Детайлна примера: Kruskal's Алгоритъм - Минимално покриващо дърво

Граф с 6 върха и 9 ребра:

graph LR A["A"] B["B"] C["C"] D["D"] E["E"] F["F"] A -->|1| B A -->|4| C B -->|2| C B -->|5| D C -->|3| D C -->|6| E D -->|7| E D -->|8| F E -->|9| F

Изпълнение на Kruskal - Стъпка по стъпка (Union-Find):

Стъп Ребро Тегло Връзка Union-Find Действие
0 - - - {A}, {B}, {C}, {D}, {E}, {F} Инициализация: всеки връх сам
1 (A,B) 1 Поне един край е нов {A,B}, {C}, {D}, {E}, {F} ✓ Добавяме. MST=1 ребро, тегло=1
2 (B,C) 2 Поне един край е нов {A,B,C}, {D}, {E}, {F} ✓ Добавяме. MST=2 ребра, тегло=3
3 (C,D) 3 Поне един край е нов {A,B,C,D}, {E}, {F} ✓ Добавяме. MST=3 ребра, тегло=6
4 (A,C) 4 Оба края във {A,B,C,D} {A,B,C,D}, {E}, {F} ✗ Пропускаме - цикъл A-B-C-A
5 (B,D) 5 Оба края във {A,B,C,D} {A,B,C,D}, {E}, {F} ✗ Пропускаме - цикъл B-D-C-B
6 (C,E) 6 Поне един край е нов (E) {A,B,C,D,E}, {F} ✓ Добавяме. MST=4 ребра, тегло=12
7 (D,E) 7 Оба края във {A,B,C,D,E} {A,B,C,D,E}, {F} ✗ Пропускаме - цикъл D-E-C-D
8 (D,F) 8 Поне един край е нов (F) {A,B,C,D,E,F} ✓ Добавяме. MST завършено! Тегло=20
9 (E,F) 9 Оба края във {A,B,C,D,E,F} - Не е необходимо - вече имаме 5 ребра

Резултат:

  • MST ребра: (A,B,1), (B,C,2), (C,D,3), (C,E,6), (D,F,8)
  • Общо тегло: 1 + 2 + 3 + 6 + 8 = 20
  • Брой ребра: 5 (което е n-1 за 6 върха) ✓
Защо Kruskal работи: Гордийската стратегия - винаги избираме най-малкото ребро, което не образува цикъл. Това гарантира оптимално решение по принципа на матроидите.
Забележка за коректност: Tanto Kruskal, както и Prim, дават оптимално решение. Доказателството използва обмен на аргумент (exchange argument).

Раздел 4: Най-кратък път

Проблем на най-краткия път

Дадени: Граф G, начален връх s, целеви връх t (или всички върхове). Намерете пътя с минимално сумарно тегло.

Алгоритъм на Dijkstra

Предпоставка: Всички тегла са неотрицателни (w(e) ≥ 0).
Dijkstra(Graph G, Vertex start): dist[start] ← 0 dist[v] ← ∞ for all other v pq ← priority queue with (0, start) while pq is not empty: (d, u) ← pq.extractMin() if d > dist[u]: continue for each neighbor v of u: if dist[u] + weight(u,v) < dist[v]: dist[v] ← dist[u] + weight(u,v) pq.insert((dist[v], v)) return dist
Сложност: O((V+E) log V) с двоична приоритетна опашка. Може да бъде O(V²) с масив.

Алгоритъм на Bellman-Ford

Преимущество: Работи с отрицателни тегла и открива отрицателни цикли!
Bellman-Ford(Graph G, Vertex start): dist[start] ← 0 dist[v] ← ∞ for all other v for i = 1 to |V|-1: for each edge (u,v,w): if dist[u] != ∞ and dist[u] + w < dist[v]: dist[v] ← dist[u] + w // Проверка за отрицателни цикли for each edge (u,v,w): if dist[u] + w < dist[v]: return "Отрицателен цикъл!" return dist
Сложност: O(V·E) - по-бавно от Dijkstra, но по-гъвкаво.

Floyd-Warshall (Всички двойки)

Проблем: Намиране на най-краткия път между ВСИЧКИ двойки върхове.
Floyd-Warshall(Matrix dist): for k = 0 to |V|-1: for i = 0 to |V|-1: for j = 0 to |V|-1: dist[i][j] ← min(dist[i][j], dist[i][k] + dist[k][j]) return dist
Сложност: O(V³) - проста, но тежка за голяма V.

Детайлна примера: Dijkstra's Алгоритъм

Претеглен граф с 5 върха, начало A:

graph LR A["A"] B["B"] C["C"] D["D"] E["E"] A -->|2| B A -->|4| C B -->|1| C B -->|7| D C -->|3| D C -->|5| E D -->|1| E

Таблица на обновяванията на разстояния:

Стъп Обработен dist[A] dist[B] dist[C] dist[D] dist[E] Описание
0 (начало) - 0 ∞ ∞ ∞ ∞ Инициализация: само A има разстояние 0
1 A (d=0) 0 2 4 ∞ ∞ Релаксирани ребра от A: A→B (2), A→C (4)
2 B (d=2) 0 2 3 9 ∞ Релаксирани ребра от B: B→C (2+1=3, по-добре), B→D (2+7=9)
3 C (d=3) 0 2 3 6 8 Релаксирани ребра от C: C→D (3+3=6, по-добре), C→E (3+5=8)
4 D (d=6) 0 2 3 6 7 Релаксирани ребра от D: D→E (6+1=7, по-добре от 8)
5 E (d=7) 0 2 3 6 7 Няма релаксации - алгоритъмът завършва

Анализ на путищата:

  • До A: A | Разстояние: 0
  • До B: A → B | Разстояние: 2
  • До C: A → B → C | Разстояние: 3 (по-добре от преката A → C = 4)
  • До D: A → B → C → D | Разстояние: 6
  • До E: A → B → C → D → E | Разстояние: 7
Ключово наблюдение: В каждом стъпе избираме върха с минимално разстояние от остатъка. Това гарантира оптималност поради свойството на релаксацията.

Сравнение на алгоритмите

Алгоритъм Сложност Отрицателни тегла Всички двойки
Dijkstra O((V+E) log V) ❌ Не Един източник
Bellman-Ford O(V·E) ✓ Да Един източник
Floyd-Warshall O(V³) ✓ Да ✓ Всички двойки

Раздел 5: Дървета (като графи)

Дърво като специален граф

Дърво: Свързан ненасочен граф без цикли. Еквивалентно: G е дърво iff G е свързан и |E| = |V| - 1.

Основни свойства

  • Единствен път: Между всеки две върхове в дърво съществува точно един прост път.
  • Ребра: За n върхове, има точно n-1 ребра.
  • Листо: Връх с степен 1 (в ненасочен случай).
  • Корен: В насочено дърво - един връх без входящи ребра (родител на всички).

Вкоренено дърво (Rooted Tree)

Вкоренено дърво: Дърво с выбран корен r. Всяко ребро е насочено от родител към дете.
  • Височина на дърво: Максимално разстояние от корена до листо.
  • Дълбочина на връх: Разстояние от корена до върха.
  • Поддърво: Дърво с корен като някой връх v.

Двоично дърво (Binary Tree)

Всеки върх има най-много 2 деца (ляво и дясно). Класификация:
  • Пълно двоично дърво: Всеки връх има 0 или 2 деца.
  • Перфектно двоично дърво: Всички листа на една височина.
  • Балансирано двоично дърво: Височина = O(log n).
За детайлно на двоични дървета като структури от данни, вж. Тема 5: Структури от данни.

Преходи на дърво

От гра́фова перспектива, преходите са специфични DFS/BFS реализации:

Вътрешен преход (Inorder) за двоично дърво

Inorder(Node v): if v is not null: Inorder(v.left) process(v) Inorder(v.right)

Преход на Euler тур

Euler тур: Тур по всички ребра точно веднъж. Възможен iff граф е свързан и всички върхове имат четна степен (за Euler верига: точно 2 върхове с нечетна степен).

Пример: Дърво с преход

Двоично дърво с 7 върха:

graph TD A["A"] B["B"] C["C"] D["D"] E["E"] F["F"] G["G"] A --> B A --> C B --> D B --> E C --> F C --> G

Преходи:

  • Inorder: D, B, E, A, F, C, G
  • Preorder: A, B, D, E, C, F, G
  • Postorder: D, E, B, F, G, C, A
  • BFS (ниво): A | B, C | D, E, F, G

Раздел 6: Специални задачи и проблеми

Euler пътя и верига

Euler верига (Euler circuit): Път, който преминава всяко ребро точно веднъж и се завършва в началния връх.
Euler пътя (Euler path): Път, който преминава всяко ребро точно веднъж (разпочива в един връх, завършва в друг).
Условие на Euler:
  • Euler верига съществува iff граф е свързан и ВСЕ върхове имат четна степен.
  • Euler пътя съществува iff граф е свързан и имаме ТОЧНО 2 върхове с нечетна степен.

Hamilton пътя и верига

Hamilton верига: Път, който посещава всеки връх точно веднъж и се завършва в началния.
Hamilton пътя: Път, който посещава всеки връх точно веднъж.
NP-пълнота: Проблемът на Hamilton верига е NP-пълен. Няма известен полиномиален алгоритъм.
Связь с Тема 3 (Алгоритми): Hamilton верига е класически пример за NP-пълен проблем, който демонстрира разликата между P и NP класове.

Оцветяване на граф (Graph Coloring)

Оцветяване на върхове: Назначаване на цветове на върхове така, че никой съседни върхове да няма същия цвят.
Хроматично число χ(G): Минимален брой цветове, необходими.

Гордийски алгоритъм на оцветяване

Greedy-Coloring(Graph G): color[v] ← undefined for all v for each vertex v in order: used_colors ← set of colors of neighbors of v color[v] ← smallest color not in used_colors return color
Сложност: O(V + E) за оцветяване.
Оптималност: Не е гарантирано оптимално решение, но е O(Δ+1)-приближение, където Δ е максимална степен.

Максимален поток / Минимално сечение (Max Flow / Min Cut)

Проблем: Дан е граф с капацитети на ребра. Намерете максимален поток от източник s до сифон t.

Ford-Fulkerson алгоритъм (кратко)

Ford-Fulkerson(Graph G, Vertex s, Vertex t): flow ← 0 while exists path from s to t in residual graph: f_path ← capacity of minimum edge in path flow += f_path update residual capacities return flow
Теорема на Max-Flow Min-Cut: Максимален поток = минимално сечение.

Двуделно съвпадение

Двуделно съвпадение (Bipartite Matching): Намиране на максимален набор от несвързващи се ребра между две раздели множества.

Може да се реши през алгоритъм на максимален поток с единични капацитети.

Теорема на König: В двуделен граф, размерът на максимално съвпадение = размерът на минимално върхово покритие.

Раздел 7: Двуделни графи (Bipartite Graphs)

Определение и основни свойства

Двуделен граф (Bipartite Graph): Ненасочен граф G = (V, E), където множеството върхове V може да бъде разделено на две непразни, непресичащи се подмножества U и W (U ∪ W = V, U ∩ W = ∅), такива че всяко ребро (u, v) ∈ E има един край в U и един край в W.

Обозначение

Двуделен граф често се обозначава като G = (U, W, E), където U и W са двете раздела.

Примери на двуделни графи

  • Пълен двуделен граф К_{m,n}: Всеки връх от U е свързан с всеки връх от W. Име: всички m+n върхове могат да бъдат разделени, така че |U|=m, |W|=n, |E|=m·n.
  • Граф на съвпаденията: Както в задачата за брак между студенти и университети.
  • Графи на проектирането: Множество хора, множество задачи, ребро означава възможност за изпълнение.
  • Социална мрежа: Двуделен граф между "намиращи" и "нашите".

Ключови теореми

Теорема 1: Характеризация на двуделни графи
Неориентиран граф G е двуделен тогава и само тогава, когато той не съдържа нечетни цикли (т.е. цикли с нечетна дължина).
Теорема 2 (König):
В двуделен граф размерът на максимално съвпадение равняется на размерът на минимално върхово покритие.
Теорема 3 (Hall's Theorem):
Нека G = (U, W, E) е двуделен граф. Съществува съвпадение, което покрива всички върхове на U тогава и само тогава, когато за всяко подмножество S ⊆ U, |N(S)| ≥ |S|, където N(S) е множеството от съседи на върховете в S във W.

Проверка за двуделност

Алгоритъм: BFS Раскрасяване

IsBipartite(Graph G): color ← empty dictionary for each unvisited vertex u in G: queue ← [u] color[u] ← 0 while queue is not empty: v ← queue.dequeue() for each neighbor w of v: if w is not colored: color[w] ← 1 - color[v] queue.enqueue(w) else if color[w] == color[v]: return false // Нечетен цикъл открит return true // Двуделен е!
Сложност: O(V + E) - същата като BFS, тъй че ще проверяваме всяка двойка съседи.

Приложения на двуделни графи

1. Съвпадение в двуделни графи (Bipartite Matching)

Проблем: Дан е двуделен граф G = (U, W, E). Намерете максимално съвпадение M ⊆ E такова, че никой два ребра от M да нямат общ край.

Решение: Може да се реши като проблем на максимален поток (max-flow) със следната техника:

  1. Добавяме източник s свързан с всички върхове от U с капацитет 1.
  2. Всяко ребро от U към W има капацитет 1.
  3. Добавяме сифон t свързан с всички върхове от W с капацитет 1.
  4. Максимален поток от s към t = размер на максимално съвпадение.

2. Покритие на върхове (Vertex Cover)

Проблем: Намерете минимално подмножество на върхове такова, че всяко ребро има поне един край в подмножеството.

König's Theorem: В двуделен граф размер на минимално върхово покритие = размер на максимално съвпадение.

Работен пример: Двуделост проверка

Граф 1: Двуделен ли е?

graph LR U1["U1"] U2["U2"] U3["U3"] W1["W1"] W2["W2"] W3["W3"] U1 --- W1 U1 --- W2 U2 --- W2 U2 --- W3 U3 --- W1 U3 --- W3

BFS Раскрасяване:

  1. Начало с U1: color[U1] = 0
  2. Съседи на U1: W1, W2 → color[W1] = 1, color[W2] = 1
  3. От W1: съсед U3 → color[U3] = 0
  4. От W2: съсед U2 → color[U2] = 0
  5. От U3: съседи W1 (вече 1), W3 → color[W3] = 1
  6. От U2: съседи W2 (вече 1), W3 (вече 1) ✓

Резултат: ДА, графът е двуделен! U = {U1, U2, U3}, W = {W1, W2, W3}

Граф 2: Двуделен ли е (с нечетен цикъл)?

graph LR A["A"] B["B"] C["C"] A --- B B --- C C --- A

BFS Раскрасяване:

  1. Начало с A: color[A] = 0
  2. Съседи на A: B, C → color[B] = 1, color[C] = 1
  3. От B: съседи A (вече 0) ✓, C (вече 1) ✓
  4. От C: съсед A (вече 0) ✓, B (вече 1) ... СТОП!
  5. Цикъл A-B-C-A има дължина 3 (нечетна)!

Резултат: НЕ, графът НЕ е двуделен - съдържа нечетен цикъл (триъгълник).

Съвпадение в двуделни графи - Примера

Двуделен граф за съвпадение:

graph LR A["Гръцко меню (A)"] B["Гръцко меню (B)"] C["Гръцко меню (C)"] X["Студент (X)"] Y["Студент (Y)"] Z["Студент (Z)"] A --- X A --- Y B --- Y B --- Z C --- X C --- Z

Максимално съвпадение:

  • Съвпадение 1: (A,X), (B,Y), (C,Z) - размер 3
  • Съвпадение 2: (A,Y), (B,Z), (C,X) - размер 3

Максимален размер: 3 (перфектно съвпадение - всички върхове покрити)

Минимално върхово покритие (König): {A, B, C} или {X, Y, Z} - размер 3

Flashcards - Интерактивни карти за повторение

Кликнете на всяка карта, за да видите отговора. Идеално за подготовка преди изпита.

Какво е граф G = (V, E)?
Наредена двойка където V е множество върхове, E ⊆ V × V е множество ребра. Може да е насочен или ненасочен.
Разлика между насочен и ненасочен граф?
Насочен (directed): ребра имат посока, (u,v) ≠ (v,u). Ненасочен (undirected): няма посока, (u,v) = (v,u).
Какво е степен на връх?
За ненасочен граф: брой ребра инцидентни на върха. За насочен: in-degree (входящи) и out-degree (изходящи).
Лема на ръкуванието
За ненасочен граф: Σ deg(v) = 2|E|. За насочен: Σ indeg(v) = Σ outdeg(v) = |E|.
Разлика между BFS и DFS?
BFS (Breadth-First): използва опашка, обхожда по нива. DFS (Depth-First): използва стек, отива възможно най-дълбоко.
Сложност на BFS/DFS?
O(V + E) при представяне със списък на съседствата. O(V²) ако се използва матрица на съседствата.
Какво е топологично сортиране?
Линейно подредяне на върхове в DAG такова, че за всяко ребро (u,v), u идва преди v. Възможно само в DAG.
Какво е MST (Минимално покриващо дърво)?
Подмножество от ребра, свързващо всички върхове с минимално сумарно тегло, без цикли. За n върхове има точно n-1 ребра.
Сложност Kruskal MST?
O(E log E) за сортиране + O(E α(V)) за Union-Find = O(E log E). Гордийски подход: мерч по растящо тегло.
Сложност Prim MST?
O((V+E) log V) с двоична приоритетна опашка, или O(V²) с масив. Разширяваме MST от един връх.
Когда се използва Dijkstra?
За намиране на най-кратък път от един източник. Предпоставка: всички тегла неотрицателни (≥0).
Сложност Dijkstra?
O((V+E) log V) с приоритетна опашка. O(V²) с масив за намиране на мин. разстояние.
Bellman-Ford срещу Dijkstra?
Dijkstra: по-бърз O((V+E)log V), но работи само за неотрицателни тегла. Bellman-Ford: O(V·E), работи с отрицателни, открива отрицателни цикли.
Floyd-Warshall сложност и приложение?
O(V³). За всички двойки най-кратки пътища. Работи с отрицателни тегла, но открива отрицателни цикли.
Какво е дърво във графовата теория?
Свързан ненасочен граф без цикли. Еквивалентно: |E| = |V| - 1 и свързан, или между всеки две върхове има точно един път.
Дърво: колко ребра за n върхове?
Точно n - 1 ребра. Това е ключово свойство на дърветата.
Какво е листо в дърво?
Връх със степен 1 (в ненасочен случай). Крайна точка на дърво без деца.
Euler верига vs Hamilton верига?
Euler: преминава ВСЯКО РЕБРО точно веднъж. Hamilton: посещава ВСЕКИ ВРЪХ точно веднъж.
Кога е възможна Euler верига?
Граф свързан И всички върхове имат четна степен. За Euler пътя: точно 2 върхове с нечетна степен.
Hamilton верига - какво да знам?
NP-пълен проблем! Няма известен полиномиален алгоритъм. Посещава всеки връх точно веднъж.
Оцветяване на граф - какво е?
Назначаване на цветове на върхове така, че никой съседни върхове да няма същия цвят. Ищем минимално число цветове (хроматично число χ(G)).
DAG определение и приложение
Directed Acyclic Graph - насочен граф без цикли. Позволява топологично сортиране. Модел за зависимости (проекти, компилаторни порядък).
Какво е двуделен граф?
Граф G = (U, W, E) където V се дели на два раздела U и W, и всяко ребро имеет един край в U и един в W. Няма ребра вътре в U или W.
Характеризация на двуделни графи
Граф е двуделен ако и само ако НЕ съдържа нечетни цикли (цикли с нечетна дължина).
Как проверяю двуделност?
BFS раскрасяване с два цвята. Ако всеки съсед има различен цвят от текущия → двуделен. Иначе → нечетен цикъл.
König's Theorem за двуделни графи
Размер на максимално съвпадение = размер на минимално върхово покритие. Специално свойство на двуделни графи!
Hall's Marriage Theorem (упрощена версия)
За двуделен граф G = (U, W) съществува съвпадение покриващо U iff за всяко подмножество S ⊆ U, |N(S)| ≥ |S|.
Матрица на съседствата: плюсове и минуси
Плюс: O(1) за проверка ребро (u,v). Минус: O(V²) памет, неефикасна за разредени графи. Добро за плътни графи.
Списък на съседствата: плюсове и минуси
Плюс: O(V+E) памет, O(deg(v)) за намиране съседи. Минус: O(deg(u)) за проверка ребро (u,v). Добро за разредени графи.
Union-Find (Disjoint Set Union)
Структура за поддържане на разделения на множество. find(x): намери представител. union(x,y): обединай множества. С path compression: O(α(n)) амортизирано.
Приоритетна опашка (Priority Queue) приложение
Използва се в Dijkstra (най-малко разстояние), Prim (най-малко ребро), Huffman кодиране. O(log n) вмъкване и извличане на минимум.
Гордийска стратегия - какво е?
Избираме най-добрия локален избор на всяка стъпка. Пример: Kruskal (минимално ребро без цикъл), Dijkstra (минимално разстояние).
